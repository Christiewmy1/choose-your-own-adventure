## What Are We Building?

A web app with two modes:

1. **Reader Mode** — Play through "The Cave of Time" interactively in the browser. Click choices, move through the story.
2. **Author Mode** — A visual tool for writing/editing branching stories. See the graph, add/edit pages, connect them.

The Fork Instructions say the reader part is "easy" and can even be static HTML. So we should nail that first and use it as the foundation, then layer the authoring tool on top.

---

## What Data Do We Have?

- `output/cot-pages-ocr-v2/` — ~115 text files, one per story page (e.g. `02-CoT.txt`)
- Each page ends with 0, 1, or 2 choices like "If you decide X, turn to page 4"
- `output/cot-story-graph.mmd` — Mermaid graph of all page connections
- `output/cot-stories/` — 45 pre-written full story paths

This means we can parse the `.txt` files in the browser (or pre-process them into JSON) and have everything we need without a backend.

---

## Tech Stack Ideas

### Option A: Pure Static HTML/JS (simplest)
- Pre-process all story pages into a single `story-data.json` file
- One `index.html` + vanilla JS
- Deploy on GitHub Pages — zero config, free
- ✅ Easy to deploy, works for everyone, no build step

### Option B: React + Vite
- Nicer component structure
- Can use a graph library like D3 or Cytoscape.js for the visual graph
- Deploy on Vercel or GitHub Pages
- ✅ Cleaner code, better for the authoring UI
- ⚠️ More setup, need Node.js

### Option C: Next.js
- Could do server-side rendering
- ❌ Overkill for this project, harder to deploy for a class assignment

**Decision: Option B (React + Vite)** — gives us clean components for the reader and graph views, and Cytoscape.js is perfect for the interactive story graph.

## Reader Mode — Ideas

### Core Flow
- Land on page 2, read the text
- See 1–2 choice buttons at the bottom
- Click a choice → navigate to next page
- Pages with no choices = endings (show a "THE END" screen)

### Nice-to-Haves
- Progress tracker: "You are on page 14 of 119 possible pages"
- Breadcrumb trail showing your path so far (e.g. 2 → 3 → 5 → 16...)
- "Back" button to undo last choice
- History sidebar showing the full path taken
- Visual indicator: are you near a dead end? a long branch?
- Highlight your current position on the graph

---

## Author Mode — Ideas

### Core Features
- Interactive graph visualization (nodes = pages, edges = choices)
- Click a node to read/edit that page's text
- See which pages are "dead ends" (terminal nodes), highlighted differently
- See which pages have no incoming edges (orphan pages)
- See the "main trunk" from page 2 highlighted

### Editing Features
- Edit the text of any page in a text area
- Add a new choice to a page (with destination page number)
- Add a new page
- Connect two pages with a new edge

### Visual Graph Ideas
- Color coding:
  - 🟢 Green = start (page 2)
  - 🔴 Red = dead end / ending pages
  - 🔵 Blue = pages on the main trunk
  - ⚪ Gray = normal pages
  - 🟡 Yellow = orphan pages (unreachable)
- Zoom + pan on the graph
- Mini-map for large graphs
- Click node → highlight all paths through it

### Export
- Export the story as a new set of `.txt` files
- Export the graph as SVG or PNG
- Download the whole story as a single HTML file (self-contained reader)

## Data Format — Pre-processed JSON

Instead of parsing `.txt` files in the browser, pre-process them into:

```json
{
  "pages": {
    "2": {
      "id": 2,
      "text": "You've hiked through Snake Canyon...",
      "choices": [
        { "text": "If you decide to start back home", "page": 4 },
        { "text": "If you decide to wait", "page": 5 }
      ],
      "isTerminal": false
    },
    "4": {
      ...
    }
  },
  "startPage": 2
}
```

This JSON can be generated by a Python script (already have the pieces) or a small Node.js script, then bundled with the app.
